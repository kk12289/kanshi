# kanshi β版 営業・ヒアリングメモ

kanshiを「日本語URL監視ツール」ではなく、Web制作・WordPress保守をしている人がクライアントサイトの異常に気づき、第一報・復旧報告まで行いやすくする道具として検証するためのメモです。

## 現在の検証仮説

- 通知が来ても、メールに埋もれて見落としてしまう
- サイトはUP判定でも、実際には表示崩れやフォーム不具合が起きている
- クライアントから連絡を受けて、初めて異常に気づく
- 原因が分からない段階で、第一報をどう送るか迷う
- 復旧後に、何をどこまで説明すればいいか悩む
- 複数のクライアントサイトを手作業で確認するのが限界

```text
突然のご連絡失礼します。
個人開発で「kanshi」という、Web制作・WordPress保守向けのURL監視ツールを作っています。

WordPress保守やホームページ更新代行の現場で、少し伺いたいことがあります。

クライアントサイトが表示されなくなったとき、
今はどのように気づいていますか？

また、障害時のクライアントへの第一報や状況説明は、
どのように対応されていますか？

kanshiでは、クライアントサイトの異常検知、公開ステータスページ、第一報・復旧報告文のコピー機能を試作しています。

β版ページはこちらです。
https://kanshi-eftn.onrender.com/beta

もし可能でしたら、
「保守業務で使えそう」
「今の内容だと使わなさそう」
のどちらかだけでも教えていただけると助かります。

よろしくお願いいたします。
内海
```
